Basic Commands

Change the Name for the Network Server

Router> enable
Router# conf t
Router(config)# hostname [Name]

This command changes the name of the router or network device, making it easier to identify and manage within a network.

Configure Console Password

R1(config)# line console 0
R1(config-line)# password console
R1(config-line)# login

This command sets a password for console access and enables login with that password.

Configure Telnet Password

R1(config)# line vty 0 4
R1(config-line)# password telnet
R1(config-line)# login

This command sets a password for remote access via Telnet and enables login with that password.

Set Enable Secret Password

R1(config)# enable secret class

Sets the privileged EXEC mode password, which is stored in a more secure, encrypted format.

Encrypt All Passwords

R1(config)# service password-encryption

This command encrypts all plain-text passwords in the configuration.

Disable DNS Lookup

R1(config)# no ip domain-lookup

Prevents the router from trying to translate incorrectly entered commands as DNS names, which can save time.

Enable CDP and View Neighbors

R1# conf t
R1(config)# cdp run
R1(config)# do show cdp neighbors

These commands enable CDP on the router and display a list of neighboring devices connected to the router.

Enhanced Interior Gateway Routing Protocol (EIGRP)

R1(config)# router eigrp [AS_number] (ex: 1, 10, 100, 111)
R1(config-router)# no auto-summary
R1(config-router)# network [network_address] [subnet_mask]

EIGRP is a Cisco proprietary routing protocol. It is one of the most popular and widely used routing protocols.

Configuring EIGRP is similar to RIP. The “1” is the AS number, and it has to be the same on all routers! We require the no auto-summary command because by default EIGRP behaves classful, and we want it to be classless.

Hot Standby Router Protocol Configuration

R1(config)# int g0/1 | int g0/1.10 | int vlan [number]
R1(config-if)# standby version 2
R1(config-if)# standby [HSRP number] ip [network address]
R1(config-if)# standby [HSRP number] priority [priority number]
R1(config-if)# standby [HSRP number] preempt

This configuration sets up Hot Standby Router Protocol (HSRP) on a router interface:

  • int g0/1 | int g0/1.10 | int vlan [number]: Selects the interface or VLAN for HSRP configuration.
  • standby version 2: Configures the HSRP to use version 2, which provides additional features over version 1.
  • standby [HSRP number] ip [network address]: Assigns the virtual IP address that clients will use as their default gateway.
  • standby [HSRP number] priority [priority number]: Sets the priority for the router in the HSRP group. Higher priority values increase the likelihood of this router becoming the active router.
  • standby [HSRP number] preempt: Enables the router to take over as the active router if it has a higher priority or becomes available after a failure.

Configure router IDs

Configure the routing process on R1 with the network statements and wildcard masks that are required to activate OSPF routing for all the attached networks. The network statement values should be the network or subnet addresses of the configured networks.

R1(config-router)# network [network_address] [wildcard_mask] area [area_id]

R2(config-router)# network 10.1.1.2 0.0.0.0 area 0
R2(config-router)# network 10.1.1.9 0.0.0.0 area 0

On router, configure the required interfaces with OSPF.

Router(config-if)# ip ospf process-id area area-id

Configure Passive Interfaces

OSPF will send its protocol traffic out of all interfaces that are participating in the OSPF process. On links that are not configured to other networks, such as LANs, this unnecessary traffic consumes resources. The passive-interface command will prevent the OSPF process from sending unnecessary routing protocol traffic out LAN interfaces.

R1(config)# router ospf [process_id]
R1(config-router)# passive-interface [interface]
R1(config)# router ospf 10
R1(config-router)# passive-interface gi0/0/0

R2(config)# router ospf 10
R2(config-router)# passive-interface gi0/0/0

R3(config)# router ospf 10
R3(config-router)# passive-interface gi0/0/0
